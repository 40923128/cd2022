AGX REPO(W4)

在fork的倉儲下，各組員pull requests組員能自行更新到原始倉儲檔案
1.新建一個自己倉儲下連接原倉儲的分支

git remote add (uptream name) (repo url.git)

(新增後可查詢自己分支細節)

git remote -v


2.拉取原倉up-to-date檔案(到自己設的分倉支)

git fetch (upstream name)

3.拉下來後，再將分支內容覆蓋至自己的原分支

git merge (upstream name)/main

上述步驟做完後就是以原倉儲上的內容進行更改
改完上傳github後，進行pull request至原倉儲做合併
